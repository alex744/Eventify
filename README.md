# Eventify

- **API для управления событиями (CRUD операции).**

## Требования

- [.NET 10 SDK](https://dotnet.microsoft.com/ru-ru/download/dotnet/10.0) 

## Запуск проекта

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/alex744/Eventify.git
   cd Eventify\src
   ```

2. Восстановите зависимости:
   ```bash
   dotnet restore
   ```

3. Соберите проект:
   ```bash
   dotnet build
   ```

4. Запустите веб‑приложение:
   ```bash
   dotnet run --project Ya.Events.WebApi
   ```

5. При необходимости запустите тесты:
   ```bash
   dotnet test
   ```

6. Приложение будет доступно по адресам:
   - `http://localhost:5000`
   - `https://localhost:5001`

7. Документация Swagger:
   - `http://localhost:5000/swagger`

## Краткая документация API

Все эндпоинты по базовому пути `/events`. 

### Модель Event

```json
{
  "id": "guid",
  "title": "string",
  "description": "string | null",
  "startAt": "2025-06-01",
  "endAt": "2025-06-02"
}
```

**Валидация:** 
- `title` обязательно.
- `endAt` позже `startAt`.

### Эндпоинты

| Метод | Путь               | Описание                                | Основные ответы                      |
|-------|--------------------|-----------------------------------------|--------------------------------------|
| GET   | `/events`          | Получить все события (с фильтрацией)    | 200: Список событий                  |
| GET   | `/events/{id}`     | Получить событие по `id` (GUID)         | 200: Событие, 404: Not Found         |
| POST  | `/events`          | Создать новое событие                   | 201: Created + заголовок Location    |
| PUT   | `/events/{id}`     | Обновить событие (полный объект)        | 204: No Content, 404: Not Found	  |
| DELETE| `/events/{id}`     | Удалить событие по `id` (GUID)          | 204: No Content, 404: Not Found      |

### Фильтрация и пагинация для GET /events

Для `GET /events` поддерживаются параметры запроса вида:
`/events?title=Конференция&from=2025-06-01&to=2025-06-30&pageSize=10&page=2`

#### Параметры фильтрации

| Параметр   | Тип        | Описание																																   |
|------------|------------|--------------------------------------------------------------------------------------------------------------------------------------------|
| `title`    | `string`   | Частичный поиск по названию события (без учёта регистра). Если не задан — фильтр не применяется.										   |
| `from`     | `datetime` | Возвращает только события, у которых `startAt` ≥ указанной даты‑времени. Если не задан — ограничение по началу интервала не накладывается. |
| `to`       | `datetime` | Возвращает только события, у которых `endAt` ≤ указанной даты‑времени. Если не задан — ограничение по концу интервала не накладывается.    |

#### Параметры пагинации

| Параметр   | Тип    | Описание																									 |
|------------|--------|--------------------------------------------------------------------------------------------------------------|
| `pageSize` | `int`  | Количество событий на одной странице (по умолчанию, 10). Если не задан — используется значение по умолчанию. |
| `page`     | `int`  | Номер страницы (начиная с 1). Если не задан — возвращается первая страница.									 |

### Примеры запросов GET /events

- Все события с названием, содержащим «Конференция»:
  `/events?title=Конференция`

- События в июне 2025 года:
  `/events?from=2025-06-01&to=2025-06-30`

- Пагинация: 10 событий, вторая страница:
  `/events?pageSize=10&page=2`

- Смешанный запрос:
  `/events?title=Доклад&from=2025-06-01&page=1`

При некорректных значениях (неверный формат даты, `from` позже `to`, отрицательные/нулевые числа в `pageSize`/`page`) API возвращает `400 Bad Request` с описанием ошибки.

### Пример POST‑запроса

```json
{
  "title": "Конференция",
  "description": "Описание (необязательно)",
  "startAt": "2025-06-01",
  "endAt": "2025-06-01"
}
```

### Формат ответов об ошибках

При ошибках валидации или бизнес‑логики возвращается `400 Bad Request` с JSON следующего вида:

```json
{
  "status": 404,
  "detail": "Событие с идентификатором '9e1514cf-90f7-46d6-9e30-8df604d59f58' не найдено."
}
```
